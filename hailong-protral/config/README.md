# 配置文件说明

本目录包含海隆咨询网站的静态配置文件。

## 文件列表

### 1. site-config.json
**用途**: 程序实际使用的配置文件（无注释，标准JSON格式）

**说明**: 
- 这是前端代码实际读取的配置文件
- 必须保持标准JSON格式（不支持注释）
- 修改此文件后，前端会自动读取新配置（无需重新编译）

### 2. site-config.jsonc
**用途**: 带详细注释的配置文件（供开发者参考）

**说明**:
- 包含每个配置项的详细中文注释
- 便于理解各配置项的含义和用途
- 修改配置时，建议先编辑此文件，然后复制到 `site-config.json`
- 注意：复制时需要删除所有注释

## 配置项说明

### 公司信息 (company)
- 公司全称、简称、英文名
- 公司口号、描述
- 成立年份、注册资本、员工数量
- Logo和网站图标路径

### 联系方式 (contact)
- 电话、邮箱、传真
- 详细地址信息（省市区街道楼层）
- 工作时间
- 地图配置（API密钥、经纬度坐标）

### 交通指引 (transportation)
- 地铁线路和站点信息
- 公交线路
- 自驾导航和停车信息
- 周边地标

### 业务范围 (businessScope)
- 业务ID、名称、图标
- 显示顺序

### 导航配置 (navigation)
- 头部导航菜单
- 页脚导航和联系信息
- 版权信息、备案号

### 常见问题 (faqs)
- 问题和答案
- 分类和显示顺序

### 社交媒体 (socialMedia)
- 微信、微博、LinkedIn等
- 启用开关和链接

### SEO配置 (seo)
- 网站标题、关键词、描述
- 作者信息

### 主题配置 (theme)
- 主色调、次要色、强调色
- 字体配置

### 功能开关 (features)
- 搜索、评论、订阅等功能
- 数据分析工具

### API配置 (api)
- 开发/生产环境API地址
- 超时时间、重试次数

### 上传配置 (upload)
- 文件大小限制
- 允许的文件类型
- 图片尺寸限制

### 分页配置 (pagination)
- 默认每页数量
- 可选的分页选项

### 缓存配置 (cache)
- 缓存开关
- 缓存时间

## 使用方法

### 修改配置的步骤

1. **编辑带注释的文件**
   ```bash
   # 打开带注释的配置文件
   code config/site-config.jsonc
   ```

2. **修改需要的配置项**
   - 根据注释说明修改对应的值
   - 保存文件

3. **复制到实际配置文件**
   ```bash
   # 方法1: 手动复制（推荐）
   # 复制 site-config.jsonc 的内容到 site-config.json
   # 删除所有注释（以 // 开头的行）
   
   # 方法2: 使用工具自动转换
   # 可以使用在线工具或编辑器插件去除注释
   ```

4. **验证JSON格式**
   - 确保 `site-config.json` 是有效的JSON格式
   - 可以使用在线JSON验证工具检查

5. **测试配置**
   - 刷新浏览器查看效果
   - 检查控制台是否有错误

### 常见修改场景

#### 修改联系电话
```jsonc
"contact": {
  "phone": "0371-55894666",  // 修改为新的电话号码
  ...
}
```

#### 修改公司地址
```jsonc
"contact": {
  "address": {
    "fullAddress": "新的完整地址",  // 修改完整地址
    ...
  }
}
```

#### 添加导航菜单
```jsonc
"navigation": {
  "header": [
    ...
    {
      "name": "新菜单",
      "path": "/new-page",
      "order": 8
    }
  ]
}
```

#### 修改地图坐标
```jsonc
"contact": {
  "map": {
    "longitude": 113.773301,  // 修改经度
    "latitude": 34.768953,    // 修改纬度
    ...
  }
}
```

## 注意事项

1. **JSON格式要求**
   - 不能有注释（实际使用的 site-config.json）
   - 字符串必须用双引号
   - 最后一个属性后不能有逗号
   - 布尔值用 true/false（小写）

2. **修改后生效**
   - 配置文件修改后会立即生效
   - 某些浏览器可能需要强制刷新（Ctrl+F5）
   - 如果使用了缓存，可能需要清除缓存

3. **备份建议**
   - 修改前建议备份原配置文件
   - 可以使用版本控制系统（Git）管理配置变更

4. **环境区分**
   - API配置支持开发和生产环境
   - 部署时会自动选择对应环境的配置
